现在的问题不是“完全没有页面”，而是多个页面已经有前端草稿，但业务口径、页面入口、状态流转、后端闭环没有完全收口。

彬姐反馈要点

当前技术交付压力大，后面开发等不及，产品设计这边需要先把页面、字段、入口、业务逻辑排清楚。不能只说“已经做了”，要能说明每个模块做到哪一步、缺什么、下一步怎么改。

五、下一步两条主线

授权和分润是前置口径，真正要继续往页面里落的就是“代理申请/审核”和“进货采购”。这两条要单独给 Figma Make 下指令。

主线 A：代理申请 / 代理审核

目标是让 D1-D5 不再只是说明文档，而是能被代理提交、平台审核、权限生效。

页面入口平台配置 / 经销商或代理管理 / 代理申请审核；也可以从模式配置里的 D1-D5 规则跳转进入。
页面结构申请列表、审核详情、资料补充、审核记录、授权生效范围。
关键字段申请人、申请 D 等级、所属区域、申请商品/品牌、门槛礼包、保证金/服务费、审核状态、授权期限。
状态流转待审核、待补资料、审核通过、审核驳回、已授权、授权到期、授权冻结。
验收口径通过后要能看出代理获得了哪些商品、哪些渠道、哪些区域、哪些收单方式的经营权限。
必须新增流程 承接 D1-D5 承接授权

主线 B：进货采购

目标是把商品从“能看见”推进到“可申请、可采购、已采购、可补货、能入库”。

页面入口代理端 / 进货采购 / 进货清单；已采购商品卡上保留“上架配置中心”入口。
页面结构商品池、收藏夹、进货清单、已采购/库存、补货、采购订单。
关键字段授权状态、采购状态、库存数量、供货方式、D 等级可见范围、进货价、CPS/代发、起订量、预计到货。
状态流转未授权、可申请、已授权、已加清单、已下单、待到货、已入库、库存不足、可补货。
验收口径收藏夹不等于进货清单，已采购不等于库存；页面要把这三个对象分开。
必须拆对象 承接订单 承接库存

代理申请/审核 Prompt

请新增或完善「代理申请 / 代理审核」页面，用来承接 D1-D5 渠道代理体系。 本次只做代理申请审核，不要改订单和库存。 页面目标： 1. 代理可以按 D1-D5 等级提交申请，平台可以审核。 2. 审核不是品牌方直接审核代理，而是平台在品牌授权范围内审核代理经营权限。 3. 审核通过后，要能看出代理获得了哪些商品、品牌、区域、渠道、收单方式和授权期限。 页面需要包含： 1. 申请列表：待审核、待补资料、已通过、已驳回、已冻结、已到期。 2. 审核详情：申请人信息、申请 D 等级、所属区域、申请商品/品牌、门槛礼包、保证金/服务费、提交资料。 3. 授权范围：可见商品、可采购商品、可上架商品、可使用渠道、可收单方式。 4. 审核操作：通过、驳回、要求补资料、冻结授权、延长授权。 5. 审核记录：操作人、操作时间、审核意见、状态变化。 注意： 1. 页面里不要出现 G1-G5、L1-L8、招商等级。 2. 文案统一用 D1-D5、平台代理授权、代理经营授权。 3. 改完请说明页面入口、改了哪些文件、是否只是前端展示。

进货采购 Prompt

请完善「进货采购」页面，把收藏、进货清单、已采购、库存、补货、采购订单这些对象拆清楚。 本次只做进货采购页面结构和状态口径，不要改模式配置和分润配置。 页面目标： 1. 代理进入进货采购后，能看到商品的授权状态、采购状态、库存状态。 2. 未授权商品只能申请平台代理经营授权，不能直接采购。 3. 已授权商品可以收藏、加入进货清单、下采购单。 4. 已采购商品要能看到采购订单、到货状态、入库状态和补货入口。 5. 已入库商品才能进入库存视角，库存不足时出现补货动作。 页面需要包含： 1. 商品池：全部可见商品，显示授权状态和 D 等级可见范围。 2. 收藏夹：只是收藏商品，不等于进货清单。 3. 进货清单：准备采购的商品，支持数量、供货方式、价格、提交采购。 4. 已采购/库存：按真实采购订单和入库结果展示，不要只按 myStock 大于 0 筛选。 5. 补货：库存不足或再次采购时生成补货单，不要只改前端数字。 6. 采购订单入口：能看到待付款、待发货、待到货、已入库、售后等状态。 注意： 1. 保留已采购商品卡上的「上架配置中心」入口。 2. 不要把收藏夹说成完整进货清单。 3. 不要把本地库存状态说成真实库存闭环。 4. 改完请说明页面入口、改了哪些文件、哪些数据仍是 mock/localStorage。
